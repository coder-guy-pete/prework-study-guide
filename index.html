Git

  • git branch to get a list of all of your branches in the repo
  • Git Cheat Sheet
  • Step by Step guide to working on code locally and then merging those changes into a remote repo:
    1. use git pull origin main to ensure that my local main branch is up to date with the remote repo main branch in Github
    2. use git status to check what branch I am are currently on
    3. use git checkout -b [branch-name] to create a new branch and switch to it or use git checkout [branch-name] to switch to a different branch
    4. make the changes that I want to make
    5. test the changes locally and update documentation in README or in comments of HTML source code accordingly
    6. use git add -A to add all of the changes I have made locally on the current branch to my next commit
    7. use git commit -m "inset commit comment" to commit the changes I have made as a new commit snapshot with a comment
    8. use git push origin [branch-name] to transmit my local branch commits to the remote repo
    9. navigate to my remote repo and check the "pull requests" tab
    10. select which branch I want to merge changes from (the branch I was working off of) which branch I want to merge to (likely the main branch, but does not have to be)
